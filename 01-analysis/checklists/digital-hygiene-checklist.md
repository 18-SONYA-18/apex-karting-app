## Галлюцинации и достоверность
- [ ] Нет выдуманных интеграций (сервисы/SDK, которых заказчик не называл)
- [ ] Нет ссылок на несуществующие стандарты / нормативы
- [ ] Все упомянутые сущности и термины есть в глоссарии
- [ ] Для «Апекс»: терминология согласована по всему проекту — **трасса / конфигурация трассы / экипировка / маршал / центр / заезд** (не «прогулка», «маршрут», «доска», «инструктор», «клуб»)
- [ ] Для «Апекс»: упомянутые API-эндпоинты существуют в спецификации `6-api/` (`auth`, `slots`, `bookings`, `profile`, `marshals`)
- [ ] Для «Апекс»: статусы брони (`active` / `cancelled` / `late_cancel` / `center_cancelled`) согласованы между моделью данных, API и UI

## Корректность требований
- [ ] ФТ и НФТ не перепутаны
- [ ] Каждое требование проверяемо (есть критерий приёмки)
- [ ] У use cases проработаны альтернативные и исключительные потоки
- [ ] Требования не противоречат друг другу и брифу заказчика
- [ ] Для «Апекс»: лимит мест на одну бронь **4** (себя + до 3 гостей, FR-12) — согласован между `SCR-004`, `LOGIC-002`, `feature-list.md`
- [ ] Для «Апекс»: правило 2 часов для ранней/поздней отмены (FR-17, FR-18) — согласовано между `SCR-006`, `BS-003`, `LOGIC-004`
- [ ] Для «Апекс`: два независимых лимита (места ≠ прокатная экипировка) — не смешиваются в расчётах (LOGIC-002)
- [ ] Для «Апекс»: идемпотентность `createBooking` через `Idempotency-Key` (R-022, NFR-9) — описана в `SCR-004`
- [ ] Для «Апекс»: сервер — источник истины по `price_total`, времени старта (UTC) и типу отмены (R-005, R-021)

## Доступность и mobile-first
- [ ] Состояния передаются не только цветом (текст/иконка/форма) — NFR-1
- [ ] Тач-зоны ≥ 44–48 pt — foundations §3.1
- [ ] Контраст на ярком солнце (контекст «у трассы») — foundations §3.2
- [ ] Layout не ломается при системном увеличении шрифта — foundations §7

## Безопасность данных
- [ ] Токены (`access_token` / `refresh_token`) хранятся в защищённом хранилище ОС (Keychain/Keystore) — NFR-11
- [ ] Клиент видит только свои данные (NFR-12) — проверено в SCR-005, SCR-006, SCR-007
- [ ] При удалении аккаунта активные брони аннулируются, прошедшие анонимизируются (R-009) — описано в SCR-007
- [ ] Контакты маршала не раскрываются клиенту (foundations §8.2) — проверено в SCR-003, SCR-006

## Как использовать

1. Пройтись по каждому пункту **до передачи в разработку**.
2. Снять галочку `[ ]` → `[x]` только если пункт подтверждён.
3. Если пункт не выполняется — завести задачу на доработку артефакта.
4. Для «Апекс» дополнительно свериться с `design-review.md` (RR-D01 — RR-D11) — там зафиксированы все решения, расходящиеся с макетом Figma.
