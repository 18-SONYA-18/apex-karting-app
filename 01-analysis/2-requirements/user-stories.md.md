# User stories

> Этап 2. Пользовательские истории. Формат: «Как <роль>, я хочу <действие>, чтобы <ценность>».
> Каждой истории — критерии приёмки.
>
> **Скоуп: клиентское приложение и API для него** (роль «Клиент»). Функции маршала и
> владельца обеспечиваются существующей инфраструктурой и здесь не описываются.

| ID | Как (роль) | Хочу (действие) | Чтобы (ценность) | Критерии приёмки | FR |
| :-- | :-- | :-- | :-- | :-- | :-- |
| US-1 | Клиент | зарегистрироваться по имени и номеру телефона без пароля | быстро начать пользоваться без барьеров | Указав имя и телефон, получаю доступ; повторный вход — по номеру телефона; сложный пароль не требуется. | FR-1, FR-2 |
| US-2 | Клиент | видеть список предстоящих слотов заездов | выбрать подходящий заезд | По умолчанию показываются ВСЕ предстоящие слоты (`only_available=false`), отсортированные по времени старта; слот показывает дату/время, конфигурацию трассы, маршала, цену, всего/свободно мест; заполненные (свободно = 0) и отменённые слоты остаются в списке с пометкой «Мест нет» и неактивной CTA записи; при отсутствии слотов — пустое состояние «Заездов пока нет». | FR-9 |
| US-3 | Клиент | фильтровать список слотов | быстро находить подходящий вариант | Могу фильтровать по дате/периоду, конфигурации трассы (новичковая/опытная), наличию свободных мест, маршалу; внутри группы (конфигурация трассы, маршал — множественный выбор) значения по «ИЛИ», между группами — по «И»; границы дат включительные, в локальной зоне; дефолт — без фильтров; пустой справочник маршалов — фильтр скрыт; активные фильтры отображаются; список обновляется по выбору. | FR-38 |
| US-4 | Клиент | открыть карточку слота со всеми параметрами | понять детали перед записью | Карточка показывает дату/время, конфигурацию трассы и тип, маршала, цену, всего/свободно мест, доступность прокатной экипировки, место сбора, геометрию трассы на карте (с fallback текстом при недоступности карты). | FR-9a, FR-44 |
| US-5 | Клиент | записаться на выбранный слот | забронировать место на заезде | После подтверждения создаётся запись; свободные места уменьшаются; запись появляется в «Моих бронированиях»; при создании передаётся `Idempotency-Key`. | FR-10, FR-45 |
| US-6 | Клиент | при записи указать свою или прокатную экипировку | приехать со своим инвентарём или взять напрокат | Можно выбрать «своя» / «прокатная» для каждого места; «прокатная» уменьшает прокатный фонд экипировки, «своя» — нет. | FR-11, FR-14 |
| US-7 | Клиент | одной записью забронировать несколько мест (себя + гости) | записать компанию за раз | Можно указать 1–4 места в одной записи при наличии свободных (клиент + до 3 гостей); для каждого места выбирается вариант экипировки; имена и контакты гостей не собираются, ответственность за гостей несёт владелец брони. | FR-12 |
| US-8 | Клиент | быть защищённым от записи сверх лимита | не приехать на занятое/несуществующее место | Лимит мест `max_seats = min(free_seats, track_config.capacity_cap, 4)` (свободные места, потолок конфигурации трассы, лимит 4 на бронь); прокатная экипировка проверяется **отдельно**: `rental_count ≤ free_rental_equipment`; при гонке запросов двойная бронь/овербукинг исключены. | FR-13, FR-15 |
| US-9 | Клиент | видеть список своих бронирований | контролировать предстоящие и прошедшие заезды | Вижу свои записи (предстоящие и прошедшие) со статусом, параметрами слота, числом мест и вариантом экипировки; пагинация `limit`/`offset`; сортировка по `slot.start_at`; разбиение на «предстоящие»/«прошедшие» — на клиенте; отображаются также отменённые брони и поздние отмены (`late_cancel`). | FR-35a |
| US-10 | Клиент | отменить свою запись до старта | освободить место, если планы изменились | Запись отменяется только целиком (частичная отмена гостя не поддерживается); при ранней отмене (≥2 ч, включая ровно 2 ч) освобождаются ВСЕ места и прокатная экипировка брони; при поздней (<2 ч) запись помечается «поздняя отмена», места не освобождаются, штрафов нет; время в UTC отображается в локальной зоне центра, статус отмены определяет сервер. | FR-16, FR-17, FR-18 |
| US-11 | Клиент | видеть цену заезда | понимать стоимость заранее | В карточке и при подтверждении показана цена (`price_total` из ответа API — источник истины); оплата офлайн (наличные/перевод), запись фиксируется без онлайн-оплаты. | FR-30, FR-45 |
| US-12 | Клиент | получать напоминание о предстоящей записи | не забыть и не пропустить заезд | Заблаговременно до старта приходит напоминание через **системный push** (нативное/гибридное приложение, APNs/FCM); тайминг — за 24 ч и за 2 ч до старта (`reminder_hours = [24, 2]`); SMS/email — Phase 2; при отказе в разрешении приложение работает без push. *(Should)* | FR-33 |
| US-13 | Клиент | просматривать ранее загруженные данные офлайн | ориентироваться у трассы при нестабильной связи | При отсутствии сети вижу ранее загруженный список слотов и свои брони с пометкой «данные могли устареть»; изменяющие действия (бронь, отмена) офлайн недоступны с понятным сообщением; единый Error/Retry для повторной загрузки; таймаут запроса ~10 с. | FR-9, FR-35a |
| US-14 | Клиент | быть защищённым от двойной брони при сбое сети | не получить дубль и не переплатить | Клиент при создании брони передаёт обязательный `Idempotency-Key`; при неопределённом результате (таймаут/обрыв) повтор с тем же ключом и телом даёт идентичный ответ и не создаёт дубль. | FR-10, FR-13 |
| US-15 | Клиент | видеть понятные сообщения об ошибках | понять причину и что делать дальше | Для ключевых операций ошибки отображаются по единой матрице (код → реакция): нехватка мест/экипировки (`slot_full`), двойная бронь (`double_booking`), отменённый/стартовавший слот (`slot_cancelled`/`slot_started`), неверный код (`invalid_code`), лимит запросов (429), серверная ошибка (5xx); текст локализован; где уместно — предлагается повтор. | FR-10, FR-16 |
| US-16 | Клиент | видеть отмену заезда центром (по погоде/организатором) | быть уведомлённым и не записаться повторно | Бронь не удаляется, а переходит в статус «Отменён центром» с причиной; приходит push-уведомление; повторная запись на этот слот запрещена (HTTP 410); кнопка записи неактивна. | FR-46 |
| US-17 | Клиент | редактировать свой профиль | поддерживать актуальные данные | Могу изменить имя и номер телефона; номер отображается замаскированным (`+7 *** *** ** 67`). | FR-47 |
| US-18 | Клиент | выйти из аккаунта | завершить сессию на устройстве | Токены очищаются из защищённого хранилища (Keychain/Keystore), переход на экран авторизации. | FR-49 |
| US-19 | Клиент | удалить аккаунт | полностью уйти из сервиса | Активные брони аннулируются и освобождают места/экипировку; прошедшие брони анонимизируются; персональные данные удаляются; переход на экран авторизации. | FR-48 |

> **Вне скоупа.** Истории маршала и владельца, а также оценки маршалов обеспечиваются
> существующей инфраструктурой и в клиентском приложении не реализуются (оценки — Phase 2, R-001).
