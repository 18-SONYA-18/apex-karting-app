## 1. Назначение и контекст

Экран показывает **все параметры одного заезда (слота)**, чтобы клиент мог принять решение
о записи и перейти к оформлению. Это промежуточный шаг между списком слотов
([SCR-002](SCR-002-slot-list.md)) и оформлением записи ([SCR-004](SCR-004-booking.md)) —
вложенный экран без таб-бара (см. [foundations §4.2](00-foundations.md#42-таб-бар-авторизованная-зона)).

Контекст использования — трасса, солнце, спешка перед заездом
([foundations §1](00-foundations.md#1-продукт-и-аудитория)): клиенту нужно быстро считать
ключевые параметры (когда, где, сколько стоит, есть ли места) и решить, записываться ли.

---

## 2. Пользователь и цель

- **Роль:** Клиент (единственная роль приложения).
- **Цель пользователя:** убедиться, что заезд подходит (время, конфигурация трассы, маршал, цена),
  и понять, есть ли свободные места, чтобы записаться.
- **Задача экрана:** дать полную и честную картину слота за один экран и предоставить
  один очевидный следующий шаг — «Записаться» — либо понятно объяснить, почему запись
  сейчас невозможна.
- **User story:** US-4 — «Как клиент, я хочу видеть все детали заезда перед записью,
  чтобы выбрать подходящий слот».

---

## 3. Навигация

**Входящие:**
- Тап по карточке слота в списке [SCR-002](SCR-002-slot-list.md).

**Исходящие:**
- **«Записаться»** (основной CTA) → [SCR-004 Оформление записи](SCR-004-booking.md).
  Активен только при наличии свободных мест (см. §6, §8).
- **Тап по карте трассы / «Открыть карту»** → шторка [BS-004 «Карта трассы»](BS-004-track-map.md)
  (открытая интерактивная карта с отрисованной трассой).
- **Назад** (в хедере / системный жест) → возврат к списку [SCR-002](SCR-002-slot-list.md).

Полная карта переходов — в [фича-листе §3](../5-mobile-app-spec/feature-list.md#3-карта-навигации).

## 4. Контент-инвентарь

Все значения приходят из данных слота/конфигурации трассы/маршала (read-only API). Лимиты и числа
**не хардкодятся** — подставляются из данных
(см. [foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия)).

| # | Поле (модель данных) | Что показываем | Обяз. | Примечание |
|---|----------------------|----------------|:-----:|------------|
| 1 | `slot.start_at` | Дата и время старта | Да | Крупно, контрастно (ключевое число). Дата + время. |
| 2 | `track_config.name` | Название конфигурации трассы | Да | Заголовок параметра «Трасса». |
| 3 | `track_config.type` | Тип конфигурации (новичковая / опытная) | Да | Бейдж/подпись рядом с названием. |
| 4 | `slot.duration_min` | Длительность заезда | Нет | Показывать, **если значение есть** в данных; при отсутствии блок скрывается. |
| 5 | `marshal.name` | Имя маршала | Да | Без контактов маршала ([foundations §8.2](00-foundations.md#82-безопасность-данных-в-ui-nfr-11-nfr-12)). |
| 6 | `slot.price` | Цена | Да | Цена за одно место (per-seat). Крупно, контрастно. |
| 7 | `slot.total_seats` / `slot.free_seats` | Всего / свободно мест | Да | Свободно — ключевое число, влияет на состояние CTA. |
| 8 | `slot.free_rental_equipment` | Свободно прокатной экипировки | Да | Показывается **числом** свободных комплектов прокатной экипировки (из данных, без хардкода); влияет на оформление в SCR-004. |
| 9 | `slot.geometry` | Карта трассы | Да | **Статичный превью Яндекс-карты с выделенной линией трассы** (компонент [foundations §4.5](00-foundations.md#45-карта-трассы-статичный-превью-яндекскарт)). Тап → шторка [BS-004](BS-004-track-map.md) (интерактивная карта). |
| 10 | `slot.meeting_point` (+ координаты) | Место сбора | Да | **Обязательный элемент.** Пин на карте трассы (§4.5) + текстовый блок (адрес/ориентир) под картой. Важно для контекста «у трассы». См. [data-model](../4-design/data-model). |

> Доступные для записи места рассчитываются ниже по сценарию как
> `min(free_seats, track_config.capacity_cap, 4)`; на этом экране показываем
> фактические `free_seats` слота, детальный расчёт лимита — на [SCR-004](SCR-004-booking.md).

## 5. Структура и иерархия

Каркас — по [foundations §4.1](00-foundations.md#41-базовый-каркас): фиксированный хедер,
скролл-контент, фиксированный нижний CTA. Таб-бар скрыт (вложенный экран).

- **Хедер (фикс.):** кнопка «Назад» + заголовок экрана (название конфигурации трассы либо «Заезд»).
- **Скролл-контент:** блоки параметров слота (§4) сверху вниз по приоритету считывания:
  дата/время → конфигурация трассы+тип (+длительность) → **карта трассы + место сбора** → маршал →
  места → прокатная экипировка → цена.
- **Нижний CTA (фикс.):** кнопка «Записаться» во всю ширину, всегда видна, не перекрыта
  контентом. Рядом — строка цены/доступности (контекст решения).

```
┌─────────────────────────────────┐
│ ‹ Назад      Карточка заезда     │  ← фикс. хедер
├─────────────────────────────────┤
│                                  │

│                                  │
│  ┌─────────────────────────────┐ │

│  │ Длинная трасса              │ │  ← track_config.name
│  │ [ Опытная ] · ~15 мин       │ │  ← track_config.type · slot.duration_min (если есть)
│  └─────────────────────────────┘ │
│  ┌─────────────────────────────┐ │
│  │ ░░ карта (превью Яндекс) ░░░ │ │  ← статичный превью трассы (§4.5)
│  │ ░╱‾╲__ линия трассы ░    │ │  ← slot.geometry + пин места сбора
│  │ [ Открыть карту › ]         │ │  ← тап → BS-004 (интерактивная карта)
│  │ 📍 Место сбора              │ │

│  └─────────────────────────────┘ │

│  │ Свободно 5 комплектов       │ │  ← free_rental_equipment (числом)
│  └─────────────────────────────┘ │
│  ┌─────────────────────────────┐ │
│  │ Цена                        │ │
│  │ 2500 ₽ за место             │ │  ← price (per-seat)

│                                  │

│        [   Записаться   ]        │  ← фикс. CTA
```

## 6. Компоненты и поведение

| Компонент | Поведение |
|-----------|-----------|
| **Кнопка «Назад»** | Возврат на [SCR-002](SCR-002-slot-list.md); тач-зона ≥ минимальной ([foundations §3.1](00-foundations.md#31-тач-зоны-и-размеры)). |
| **Блоки параметров** | Карточки/строки «лейбл + значение». Ключевые числа (дата/время, цена, свободно мест) — крупные и контрастные ([foundations §3.2](00-foundations.md#32-контраст-и-читаемость-nfr-1)). Каждый блок — самостоятельная единица чтения. |
| **Блок «Тип конфигурации трассы»** | Бейдж с текстовой подписью (не только цветом — [foundations §3.2](00-foundations.md#32-контраст-и-читаемость-nfr-1)). |
| **Блок «Длительность»** | Отображается только при наличии `slot.duration_min`; при отсутствии блок скрывается (без «прочерков»). |
| **Блок «Карта трассы»** | Статичный превью Яндекс-карты с выделенной линией трассы (`slot.geometry`) и пином места сбора — компонент [foundations §4.5](00-foundations.md#45-карта-трассы-статичный-превью-яндекскарт). Тап → шторка [BS-004 «Карта трассы»](BS-004-track-map.md) (интерактивная карта). Состояния и fallback — по §4.5. |
| **Блок «Место сбора»** | **Обязательный.** Текстовый блок (адрес/ориентир, `slot.meeting_point`) под картой + пин на карте. Служит текстовым эквивалентом карты (§9). |
| **Блок «Прокатная экипировка»** | Показывает **число свободных комплектов прокатной экипировки** из данных, без хардкода (см. §8). |
| **CTA «Записаться» (enabled)** | При `free_seats > 0` — активна, переход на [SCR-004](SCR-004-booking.md). Во всю ширину, высота ≥ тач-зоны. |
| **CTA «Записаться» (disabled)** | При `free_seats = 0` — неактивна, с видимой пояснительной пометкой «Мест нет» (не только цвет/прозрачность — дублируется текстом). Тап не выполняет переход. |

## 7. Состояния экрана

Базовый паттерн — [foundations §5](00-foundations.md#5-сквозной-паттерн-состояний-экрана).
Ниже — специфика SCR-003.

| Состояние | Содержимое / Поведение |
|-----------|------------------------|
| **Loading** | Скелетон в форме будущего контента (блоки параметров + место под CTA). Не пустой экран ([foundations §5](00-foundations.md#5-сквозной-паттерн-состояний-экрана)). |
| **Content** | Все параметры слота (§4) + активный CTA «Записаться». Основной сценарий. |
| **Error** | Заглушка ошибки + кнопка **«Обновить»**. Текст — общий из [foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия): «Не удалось загрузить. Проверьте соединение и попробуйте снова.» |
| **Нет свободных мест** (`free_seats = 0`) | Все параметры отображены; CTA **неактивна** с пометкой «Мест нет». Спокойный тон, без давления (P6). |
| **Слот отменён / недоступен** | Параметры показаны с явной пометкой «Заезд отменён» / «Слот недоступен»; CTA «Записаться» **неактивна** или скрыта. Объяснение причины нейтральным тоном. |
| **Карта: загрузка / ошибка / офлайн** | По правилам [foundations §4.5](00-foundations.md#45-карта-трассы-статичный-превью-яндекскарт): skeleton при загрузке; при ошибке/офлайн — **fallback на текстовый блок места сбора** + ссылка «Открыть в Яндекс.Картах». Остальные параметры и CTA «Записаться» остаются доступны. |

## 8. Валидации и микрокопия

Тон и общие тексты — [foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия).

- **CTA «Записаться» неактивна при `free_seats = 0`.** Пометка рядом/на кнопке: «Мест нет».
  Никаких штрафов и упрёков (P6).
- **Доступность мест:** формулировка вида «Свободно `{free_seats}` из `{total_seats}`» —
  числа подставляются из данных слота, не хардкодятся.
- **Доступность прокатной экипировки:** показывается **числом** свободных комплектов прокатной экипировки
  (`slot.free_rental_equipment`, из данных, без хардкода); лейблы экипировки — «Своя экипировка» /
  «Прокатная экипировка» ([foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия)).
  Точный расчёт лимита `min(free_seats, track_config.capacity_cap, 4)` применяется на
  [SCR-004](SCR-004-booking.md).
- **Место сбора (обязательно):** текстовый блок с адресом/ориентиром (`slot.meeting_point`)
  под картой + пин на карте трассы. Например: «Место сбора: картинг-центр «Апекс», въезд 1».
- **Карта трассы:** статичный превью Яндекс-карты с выделенной линией трассы и пином места
  сбора ([foundations §4.5](00-foundations.md#45-карта-трассы-статичный-превью-яндекскарт));
  тап / «Открыть карту» → шторка [BS-004](BS-004-track-map.md) с интерактивной картой. Текст места
  сбора дублирует карту (доступность, §9).
- **Цена:** показывается как цена **за одно место** (per-seat), значение `slot.price`
  из данных. Итоговая сумма за все места считается на [SCR-004](SCR-004-booking.md).
- **Напоминание об офлайн-оплате:** текст из
  [foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия) —
  «Оплата на месте: наличные или перевод на карту.»

## 9. Доступность и mobile-first

Полные правила — [foundations §7](00-foundations.md#7-доступность-nfr-1),
[§3.1](00-foundations.md#31-тач-зоны-и-размеры),
[§3.2](00-foundations.md#32-контраст-и-читаемость-nfr-1).

- Ключевые числа (дата/время старта, цена за место, свободно мест) — крупные и контрастные,
  читаемы на ярком солнце.
- CTA «Записаться» — крупный, во всю ширину контентной области, тач-зона ≥ минимальной.
- Состояния (тип конфигурации трассы, «Мест нет», «отменено») не передаются только цветом —
  дублируются текстом/иконкой/формой.
- **Карта трассы не единственный носитель:** место сбора и трасса продублированы текстом
  (адрес/ориентир + название и тип конфигурации трассы); карта имеет доступное имя; при недоступности карты
  текстовый блок места сбора остаётся ([foundations §4.5, §7](00-foundations.md#45-карта-трассы-статичный-превью-яндекскарт)).
- Все интерактивные элементы имеют доступное имя; layout не ломается при системном
  увеличении шрифта.

## 10. Критерии приёмки дизайна

```gherkin
AC-001: Отображение всех параметров слота
  Дано клиент открыл карточку слота со свободными местами
  Тогда на экране показаны дата и время старта, название и тип конфигурации трассы,
    имя маршала, цена за место, всего и свободно мест и доступность прокатной экипировки
  И длительность заезда показана, если значение есть в данных
  И показаны карта трассы и место сбора

AC-008: Карта трассы и место сбора
  Дано клиент открыл карточку слота
  Тогда показан статичный превью Яндекс-карты с выделенной линией трассы и пином места сбора
  И под картой показан текстовый блок места сбора (адрес/ориентир)
  Когда клиент тапает по карте или «Открыть карту»
  Тогда открывается шторка BS-004 с интерактивной картой и отрисованной трассой

AC-009: Карта недоступна — текстовый fallback
  Дано карта не загрузилась (ошибка/офлайн/нет ключа)
  Тогда вместо карты показан текстовый блок места сбора и ссылка «Открыть в Яндекс.Картах»
  И остальные параметры слота и кнопка «Записаться» остаются доступны

AC-002: Переход к оформлению при наличии мест
  Дано слот имеет free_seats больше нуля
  Тогда кнопка «Записаться» активна
  Когда клиент нажимает «Записаться»
  Тогда происходит переход на экран оформления записи (SCR-004)

AC-003: CTA неактивна при отсутствии мест
  Дано слот имеет free_seats равное нулю
  Тогда кнопка «Записаться» неактивна
  И рядом с кнопкой показана пометка «Мест нет»
  И тап по кнопке не выполняет переход

AC-004: Возврат к списку
  Дано клиент находится на карточке слота
  Когда клиент нажимает «Назад»
  Тогда происходит возврат к списку слотов (SCR-002)

AC-005: Состояние ошибки загрузки
  Дано запрос данных слота завершился ошибкой
  Тогда показана заглушка ошибки и кнопка «Обновить»
  Когда клиент нажимает «Обновить»
  Тогда выполняется повторный запрос данных слота

AC-006: Отменённый или недоступный слот
  Дано слот отменён или недоступен для записи
  Тогда показаны параметры слота с пометкой о причине
  И кнопка «Записаться» неактивна или скрыта

AC-007: Напоминание об офлайн-оплате
  Дано клиент видит блок цены
  Тогда показано напоминание «Оплата на месте: наличные или перевод на карту.»
```

## 11. Решения

- **Длительность заезда:** показывается **при наличии** `slot.duration_min`; при отсутствии блок скрывается (§4, §6).
- **Прокатная экипировка:** показывается **числом** свободных комплектов прокатной экипировки (`slot.free_rental_equipment`) (§6, §8).
- **Описание конфигурации трассы:** **вне MVP** — отдельного текстового блока описания нет; на карточке название, тип и карта.
- **Место сбора:** **обязательный элемент** — пин на карте трассы + текстовый блок (адрес/ориентир, `slot.meeting_point`). См. [data-model](../4-design/data-model).
- **Трасса — карта:** статичный превью Яндекс-карты с выделенной линией трассы (`slot.geometry`); тап → шторка [BS-004](BS-004-track-map.md) с интерактивной картой. Компонент — [foundations §4.5](00-foundations.md#45-карта-трассы-статичный-превью-яндекскарт). Ключ API Яндекс.Карт — параметр конфигурации.
